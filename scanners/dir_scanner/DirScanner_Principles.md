# DirScanner 目录扫描工具 - 原理说明

## 概述

目录扫描（Directory Brute-forcing）是 Web 渗透测试中最基础的信息收集手段。通过向 Web 服务器发送大量路径请求，根据响应状态码和内容判断哪些路径真实存在，从而发现隐藏的后台、配置文件、备份文件等敏感资源。

---

## 核心特性

### 1. 递归扫描 (Recursive Scan) — 借鉴 dirmap

传统工具（如 dirsearch）只在发现的目录下继续扫描。本工具的区别：

| 策略 | 行为 | 优势 |
|:-----|:-----|:-----|
| dirsearch | 父目录 404 → 不再扫描子目录 | 请求少，速度快 |
| **本工具** | 父目录 404 → **仍扫描子目录** | 发现更多隐藏路径 |

**为什么 404 的子目录可能存在？**
- 服务器可能对目录路径返回 404（未配置目录索引），但子路径实际映射到具体文件或路由
- 例如 `/api/` 返回 404，但 `/api/v1/users` 返回 200
- Spring Boot 的 `/actuator/` 可能 404，但 `/actuator/health` 是 200

### 2. 自定义 404 检测

许多服务器将对不存在路径的请求重定向到自定义 404 页面，但 HTTP 状态码返回 200。本工具：

1. 发送多个随机不存在路径
2. 如果全部返回 200，提取 404 页面特征（内容长度、hash、关键词）
3. 后续扫描中，将匹配 404 特征的 200 响应判定为"不存在"

### 3. 分类字典设计

按目标类型选择对应字典，避免无效请求：

| 字典 | 覆盖范围 | 条目数 |
|:-----|:---------|:------:|
| `common.txt` | 通用目录、后台、配置、备份 | ~180 |
| `oa.txt` | 泛微/致远/用友/蓝凌/通达等 OA 系统 | ~100 |
| `cms.txt` | WordPress/Joomla/Drupal/DedeCMS/Discuz 等 | ~150 |
| `framework.txt` | Spring/Laravel/Django/Flask/Rails/Next.js 等 | ~130 |
| `middleware.txt` | Tomcat/Nginx/Apache/WebLogic/JBoss/Solr 等 | ~170 |
| `sensitive.txt` | Git/SVN/凭证/密钥/备份/日志/IDE 文件 | ~190 |
| `api.txt` | RESTful/GraphQL/WebHook/常见 API 路由 | ~160 |

### 4. 后缀扩展

目录路径自动追加常见文件后缀，发现更多隐藏文件：

```
admin → admin/, admin/index.html, admin/index.php, admin/login.php, ...
config → config.php, config.json, config.yaml, config.php.bak, ...
```

## 扫描流程

```
输入 URL + 字典
    │
    ▼
检测自定义 404 ──→ 建立 404 特征基线
    │
    ▼
层级 0: 扫描 /根目录
    ├─ [200] /admin    → 加入递归队列
    ├─ [403] /config   → 加入递归队列
    ├─ [404] /api      → 加入递归队列 (★ 与 dirsearch 不同)
    └─ [200+404特征]   → 判定为 404, 不加入队列
    │
    ▼
层级 1: 扫描 /admin/, /config/, /api/
    ├─ [200] /admin/login.php
    ├─ [200] /api/v1/users
    └─ ...
    │
    ▼
层级 2: 继续递归... (直到 max-depth)
    │
    ▼
结果汇总 (按状态码分组)
```

## 状态码含义

| 状态码 | 含义 | 关注度 |
|:-------|:-----|:------:|
| 200 | 资源存在，可直接访问 | 高 |
| 301/302 | 重定向，目录可能存在 | 高 |
| 401 | 需要认证，路径存在 | 高 |
| 403 | 禁止访问，路径存在 | 高 |
| 404 | 路径不存在 (或自定义 404) | 低 |
| 500 | 服务器错误，路径可能触发异常 | 中 |

## 防御建议

| 措施 | 说明 |
|:-----|:-----|
| **禁止目录列表** | 关闭 `autoindex`，避免目录内容泄露 |
| **统一错误页面** | 所有不存在路径返回相同 404 页面，不泄露路径信息 |
| **速率限制** | 配置 WAF 限制单 IP 请求频率，阻断爆破 |
| **IP 黑名单** | 检测异常扫描行为后封禁 IP |
| **最小暴露面** | 删除不必要的后台入口、调试接口、备份文件 |
| **访问控制** | 敏感路径加认证，不对外网开放 |
