# Multi-Head Attention 多头注意力机制 — 原理说明

## 核心概念

多头注意力（Multi-Head Attention）将 Q/K/V 投影到 $h$ 个不同的低维子空间，**每个头独立计算注意力**，最后拼接所有头的输出并融合。不同头可以关注不同的模式：一个头关注语法依赖，另一个关注语义关联，又一个关注位置关系。

---

## 数学公式

### 单头注意力

$$\text{head}_i = \text{Attention}(QW_i^Q, \; KW_i^K, \; VW_i^V)$$

其中：
- $W_i^Q \in \mathbb{R}^{d \times d_k}$，$W_i^K \in \mathbb{R}^{d \times d_k}$，$W_i^V \in \mathbb{R}^{d \times d_v}$
- $d_k = d_v = d / h$，每个头的子空间维度

### 多头拼接与输出投影

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \text{head}_2, \ldots, \text{head}_h) \; W^O$$

- $W^O \in \mathbb{R}^{hd_v \times d}$：输出投影矩阵，将拼接后的 $hd_v$ 维映射回 $d$ 维
- 最终输出维度与输入相同，便于残差连接

### 完整计算流程

```
输入 X ∈ ℝⁿˣᵈ
    │
    ├── Q = XW^Q ──┐
    ├── K = XW^K ──┤
    └── V = XW^V ──┘
          │
          ▼  沿 d 维度切分为 h 个头
    ┌─────┴─────┐
    │  head_1   │  Attention(Q₁, K₁, V₁)  ∈ ℝⁿˣᵈᵏ
    │  head_2   │  Attention(Q₂, K₂, V₂)  ∈ ℝⁿˣᵈᵏ
    │   ...     │
    │  head_h   │  Attention(Qₕ, Kₕ, Vₕ)  ∈ ℝⁿˣᵈᵏ
    └─────┬─────┘
          │
          ▼  Concat → ℝⁿˣ⁽ʰ·ᵈᵏ⁾
          │
          ▼  × W^O → ℝⁿˣᵈ
          │
          ▼  残差连接 + LayerNorm
          │
        输出
```

## 为什么需要多头

单头注意力只能学习一种注意力模式，表达能力受限。多头注意力的优势：

| 维度 | 单头 | 多头 |
|:-----|:-----|:-----|
| **子空间** | 全空间单一投影 | $h$ 个独立子空间 |
| **模式多样性** | 只学一种关联模式 | 不同头学不同模式 |
| **表达力** | 受限于单投影 | 指数级增强（理论） |
| **实验效果** | 基线 | 显著优于单头 |

**实验观察**（以机器翻译为例）：
- 某些头专注于**相邻词**的局部依赖
- 某些头专注于**长距离**的语法关系（如主语-谓语）
- 某些头学到**位置偏移**模式（总是关注前第 $k$ 个词）
- 某些头学到**语义关联**（代词与指代对象）

## 头数与模型维度的关系

$$d_{\text{model}} = h \times d_k$$

| 模型 | $d_{\text{model}}$ | $h$ (头数) | $d_k$ (每头维度) |
|:-----|:-------------------|:-----------|:-----------------|
| Transformer-Base | 512 | 8 | 64 |
| Transformer-Large | 1024 | 16 | 64 |
| BERT-Base | 768 | 12 | 64 |
| BERT-Large | 1024 | 16 | 64 |
| GPT-2 | 768 | 12 | 64 |
| GPT-3 | 12288 | 96 | 128 |
| LLaMA-7B | 4096 | 32 | 128 |

**关键约束**：$d_k$ 必须整除 $d_{\text{model}}$，且 $d_k$ 通常取 64 或 128。$d_k$ 过小单头表达力不足，过大则头数少、模式多样性下降。

## 自注意力模式 vs 交叉注意力模式

多头注意力既可用于自注意力，也可用于交叉注意力：

| 模式 | Q 来源 | K/V 来源 | 典型位置 |
|:-----|:-------|:---------|:---------|
| **Multi-Head Self-Attention** | 同一输入 $X$ | 同一输入 $X$ | Encoder、Decoder 自注意力层 |
| **Multi-Head Cross-Attention** | Decoder 输入 $Y$ | Encoder 输出 $X$ | Decoder 交叉注意力层 |

实现上代码结构完全相同，区别仅在于传入的 Q/K/V 来源不同。

## 掩码类型

| 掩码 | 形状 | 作用 | 应用场景 |
|:-----|:-----|:-----|:---------|
| **因果掩码（Causal Mask）** | $(n, n)$ 上三角 $-\infty$ | 防止看到未来位置 | GPT 等自回归模型 |
| **填充掩码（Padding Mask）** | $(n,)$ 或 $(n, n)$ | 忽略 padding 位置 | BERT 等批量处理变长序列 |

填充掩码实现：将 padding 位置的 K 行加上 $-\infty$，使其注意力权重为 0。

---

## 残差连接 + LayerNorm

多头注意力的输出经过残差连接和层归一化：

$$\text{Output} = \text{LayerNorm}(X + \text{MultiHead}(Q, K, V))$$

- **残差连接**：梯度可直接回传到输入，缓解深层网络的梯度消失
- **LayerNorm**：对每个位置的特征归一化到均值 0、方差 1，稳定训练

---

## "Attention Is All You Need" 论文关键结论

1. **多头优于单头**：8 头比单头 BLEU 提升约 +1.5
2. **头数与性能**：头数 8 是最优平衡点（论文实验），头数过多反而下降
3. **注意力可视化**：不同头学到截然不同的关注模式
4. **缩放因子必要**：去除 $1/\sqrt{d_k}$ 后性能显著下降

## 实践中头数选择的经验

| 建议 | 说明 |
|:-----|:-----|
| **$d_k = 64$ 是经验最优** | 几乎所有主流模型都采用 64 或 128 |
| **头数随模型增大而增大** | 小模型 8-12 头，大模型 32-96 头 |
| **更多头 ≠ 更好** | 头数过多时每个头维度过小，单头表达力不足 |
| **GQA 折中** | LLaMA-2 用分组查询注意力，减少推理时 KV Cache 开销 |
